## Local Interpretation

This comment is important because it reports relaunches after wake-from-sleep on `2.0b15` even when spacing settings are the same on all displays.

That does not necessarily contradict PR #691, because:

- the comment is about `2.0b15`, while PR #691 was merged after the issues and may target a later beta/release;
- the user describes wake-from-sleep with two external displays, not just explicit connect/disconnect;
- the claim "same spacing settings on all displays" is user-reported and may still need code/log verification.

But it does weaken the earlier model that equal spacing fully avoids the relaunch wave in all cases. The current safer framing is: equal spacing can avoid relaunches in the known mismatch path, but wake/sleep/display reattachment may still trigger relaunches in some configurations or builds.
