## PR Body Summary

The PR says it adds:

- just-in-time confirmation before menu bar spacing changes relaunch apps;
- override toggle to disable relaunch confirmations;
- confirmation settings persisted per profile;
- reworded relaunch warnings that state data-loss risk plainly.

It explicitly describes the old behavior as:

- automatic display-transition path applies different spacing and relaunches without warning;
- in-pane Apply/global confirmations described the relaunch as brief/harmless.

It explicitly describes the new behavior as:

- `DisplaySettingsManager.applyActiveDisplaySpacing` presents app-modal `NSAlert` before relaunching on display transitions;
- prompt is gated by `MenuBarItemSpacingManager.willRelaunch(forOffset:)`, so no no-op prompt;
- declining keeps current on-disk spacing and leaves `lastAppliedActiveDisplayUUID` untouched so next genuine transition re-prompts;
- `confirmSpacingRelaunch` toggle defaults on;
- `"Don't ask again"` suppression checkbox disables future confirmations;
- `unconfirmedSpacingProfileScope` controls whether unconfirmed applies save to active profile or all profiles.
