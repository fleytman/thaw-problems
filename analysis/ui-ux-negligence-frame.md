## Useful Frame

The old UI/UX negligence packet frames the broader problem as a failure to protect users' work: software loses drafts, state, context, diagnostic traces, or active user effort because preservation is not treated as a default responsibility.

For the Thaw issue, the best transfer is narrow:

- the harm is not merely visual annoyance;
- the relaunch wave can destroy active work in unrelated apps;
- this is `user-work loss`, not just a menu bar spacing implementation detail;
- routine display reconnect becomes a `destructive default`;
- the user pays a `redo cost` when typed input/state is lost;
- if relaunch is technically necessary, it should be explicit, predictable, and user-controlled.

## Terms Worth Using In Public Comment

- `protect users' work`
- `user-work loss`
- `destructive defaults`
- `recoverable state`
- `redo cost`
- `interface antipattern`

## Terms To Avoid Or Use Carefully

- `negligence` can sound legally/morally escalatory; useful internally, but maybe too aggressive for the GitHub comment.
- `dark pattern` is not the right label because the issue is likely not intentional deception.
- `critical bug` is acceptable if tied to concrete impact: lost typed input and unexpected app restarts.
